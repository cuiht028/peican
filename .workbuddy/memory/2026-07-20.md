## Bug 修复：flet 0.86 兼容性

用户安装 flet 0.86.1 后运行报错 `ColorScheme.__init__() got an unexpected keyword argument 'background'`。

根因：flet 0.80+ 的 `ColorScheme` 移除了 `background` 参数（Material 3 改用 `surface`）。

修复（main.py）：
- `ColorScheme(background=...)` → `ColorScheme(surface=...)`
- `ft.app(target=main)` → `ft.run(main)`（0.80+ 废弃 app()）
- `page.go()` 保留（0.90.0 前仍可用，异步 push_route 替换需大幅重构）

验证：app 成功启动无报错，24/24 测试通过。

## Bug 修复：page.go() 不触发 on_route_change（界面仍空白）

上一轮修复后用户仍报白屏。调试发现 `page.go("/")` 在 flet 0.86 中用 `asyncio.create_task(push_route())` 异步调度，在 `main()` 初始化阶段事件循环未就绪，导致 `on_route_change` **从未被调用**，视图未添加到 `page.views`。

修复（main.py）：
- 移除 `page.go("/")`，改为直接调用 `on_route_change(page, "/")` 构建初始视图
- `on_route_change` 增加可选 `route` 参数，支持手动指定路由
- 事件处理器改为 `lambda e=None: ...` 兼容 flet 0.86 无参数调用约定
- 页面内导航仍用 `page.go()`（从按钮点击事件中调用时事件循环已就绪，正常工作）

验证：调试输出确认 view 构建成功 + views count=1 + page.update() 调用，24/24 测试通过。

## Bug 修复：保存/删除后页面不刷新（成员数仍为0）

根因：`page.go("/member")` 在已处于 `/member` 页面时路由不变，`on_route_change` 不触发，页面不刷新。flet 0.86 的 `page.go()` 用 `asyncio.create_task` 异步调度，同路由不触发事件。

修复：所有同路由刷新改为直接重建视图（`page.views.clear()` → `page.views.append(xxx_page(page))` → `page.update()`），共 7 处：
- member_page.py: 保存成员后、删除成员后
- template_page.py: 保存模板后、重命名后、应用模板后、删除模板后
- result_page.py: 刷新配餐后

验证：24/24 测试通过，语法检查通过。

## Bug 修复：对话框无法关闭（flet 0.86 对话框 API 全面修正）

根因：flet 0.86 有专门的对话框管理系统 `page.show_dialog()` / `page.pop_dialog()`，不应使用 `page.overlay` 管理对话框。之前用 `page.overlay.append/remove` + `dlg.open = True/False` 方式不生效。

全面修正（一次性修复所有对话框）：
- 打开对话框：`page.overlay.append(dlg)` + `dlg.open = True` → `page.show_dialog(dlg)`
- 关闭对话框：`dialog.open = False` + `page.overlay.remove(dialog)` → `page.pop_dialog()`
- SnackBar：`page.overlay.append(sb)` + `sb.open = True` → `page.show_dialog(sb)`

影响文件：member_page.py(4 处对话框)、template_page.py(2 处对话框)、theme.py(1 处 SnackBar)
全项目已无任何 `page.overlay`/`page.dialog`/`.open = True/False` 旧用法。

验证：app 启动正常，24/24 测试通过。

注意：`pop_dialog()` 仅调用 `dialog.update()` 可能不同步全页状态。最终方案：
- 打开：`page.show_dialog(dlg)`（flet 0.86 正确注册对话框）
- 关闭/保存/删除：`dlg.open = False` + 重建 views + `page.update()`（一次性同步全页状态含对话框）
- 取消：`dlg.open = False` + `page.update()`
- SnackBar：`sb.open = True` + `page.show_dialog(sb)` + `page.update()`
